## アニメーション設計

### 基本的な仕組み

Pyxel では `pyxel.frame_count` を使って「何フレーム目か」を取得できる。
これをフレーム数で割った余りでアニメーションフレームを切り替える。

```python
# 毎秒4回切り替わる2フレームアニメーション（30fps前提）
anim_frame = (pyxel.frame_count // 8) % 2
```

スプライトシートに複数フレームを横に並べ、`blt` の `u` 座標をずらして描画する。

---

### 各アニメーションの設計

#### ① プレイヤー歩行アニメーション（フィールド）

- **フレーム数**: 2フレーム（足の開閉）
- **切り替え速度**: 8フレームごと（移動中のみ）
- **スプライト配置**（バンク1）:

```
向き   フレーム0    フレーム1
下     ( 0,  0)    (16,  0)   ← 現在フレーム0のみ実装
上     ( 0, 16)    (16, 16)
左     ( 0, 32)    (16, 32)
右     ( 0, 48)    (16, 48)
```

- **コード変更箇所**: `scenes/field_scene.py` の `draw()` 内 `blt` 呼び出し
  - `u = (frame_count // 8 % 2) * 16` で横オフセットを切り替え
  - 移動方向を `self._last_dir` として保持し `v` を選択

---

#### ② モンスター待機アニメーション（バトル）

ゲームボーイ風の「ふわふわ揺れ」や「点滅」は **スプライトを使わずコードで実現** できる。

| 演出 | 実装方法 |
|------|---------|
| 上下に揺れる | `y_offset = sin(frame_count * 0.1) * 2` 程度の揺れ |
| 点滅（ダメージ時） | `frame_count % 4 < 2` で描画をスキップ |

スプライト2フレームアニメにする場合は、バンク1の各モンスター座標を以下に変更：

```
モンスター1体 = 128×64（フレーム0が左64px、フレーム1が右64px）
→ 4列構成は維持しつつ横に2倍のスペースが必要なため、バンク構成の見直しが必要
```

**推奨**: まず1フレームで作成し、後からフレーム追加（バンク2を活用）。

---

#### ③ エンカウント演出（フィールド → バトル）

スプライトなし・コードのみで実現可能。

```
白フラッシュ → 画面を数フレーム白で塗りつぶす
交互フラッシュ → 白黒を交互に4回点滅させた後バトル画面へ
```

- `SceneManager.switch()` の前に `flash_timer` を設け、フラッシュ中は画面を `pyxel.cls(7)` で白塗り
- タイマーが切れたら実際にシーン遷移

---

#### ④ テキスト送り演出（バトルメッセージ）

現在は全文を一度に表示している。タイプライター風にする場合：

- `msg_char_count` カウンターを持ち、フレームごとに +1
- `message[:msg_char_count]` を表示
- `btnp(KEY_CONFIRM)` で全表示にスキップ

---

#### ⑤ HP バーアニメーション

ダメージ後に現在値がアニメーションしながら減少する演出。

- `displayed_hp` と `target_hp` を別々に保持
- 毎フレーム `displayed_hp` を `target_hp` に近づける（1〜2HP/フレーム）

## 制作の進め方

1. **最優先**：モンスタースプライト 15体（バンク1、各64×64）を作成
2. `create_assets.py` の `main()` 内に `_draw_monster(img1, spec_id, u, v)` を追加
3. `uv run python create_assets.py` で再生成
4. `scenes/battle_scene.py` の `rectb` → `blt` に切り替え
5. コードのみで実現できるアニメーション（フラッシュ・揺れ・点滅）を追加
6. タイルセット・マップを徐々に改善

> Pyxelエディタ（`uv run pyxel edit assets/game.pyxres`）で直接描くのが最も直感的。
